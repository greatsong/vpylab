# 학생 프로젝트 진행 시뮬레이션

작성일: 2026-05-08

이 문서는 VPyLab에서 학생이 프로젝트를 시작하고, 저장 이력을 남기고, 마지막에 소개글로 정리하는 과정을 보여주기 위한 본보기입니다. 실제 수업에서는 학생 이름과 작품 주제만 바꾸어 그대로 시연 자료로 사용할 수 있습니다.

## 1. 시뮬레이션 목표

- 새 프로젝트를 만들면 바로 작업 공간이 열리는지 확인한다.
- GitHub 저장소와 Pages 실행 페이지가 뒤에서 준비되는 동안에도 학생이 코드를 계속 쓸 수 있는지 확인한다.
- 저장 모달의 빠른 저장과 자세히 저장이 항해 일지로 자연스럽게 이어지는지 확인한다.
- 마지막에 README 소개글을 쓸 만큼 충분한 기록이 남는지 확인한다.

## 2. 본보기 프로젝트

| 항목 | 내용 |
| --- | --- |
| 프로젝트 이름 | 무지개 궤도 정원 |
| 한 줄 설명 | 여러 색의 공이 원형 궤도를 돌고, 중심에는 빛나는 정원이 있는 3D 작품 |
| 시작 방식 | 샌드박스에서 간단한 공 하나를 만든 뒤 새 프로젝트로 저장 |
| 참여 방식 | 1인 프로젝트로 시작하되, 중간에 친구가 합류해 색과 움직임을 나누어 작업할 수 있는 구조 |
| 최종 산출물 | `main.py`, GitHub Pages 실행 페이지, 항해 일지, README 소개글 |

이름 처리 원칙:

- 학생에게 보이는 프로젝트 이름은 `무지개 궤도 정원`처럼 한글을 그대로 쓴다.
- GitHub 저장소 이름은 링크 호환성을 위해 `vpylab-project-xxxxxx`처럼 영문/숫자 slug로 자동 생성된다. 필요하면 프로젝트 생성 화면에서 직접 수정할 수 있다.
- README 제목, Pages 제목, 항해 일지에는 한글 프로젝트 이름이 그대로 남는다.

## 3. 학생 시연 흐름

### 3.1 샌드박스에서 첫 장면 만들기

학생은 먼저 샌드박스에서 아주 작은 장면을 만든다. 아직 완성하려고 하지 않고, 프로젝트 이름을 붙일 만한 씨앗만 만든다.

```python
from vpython import *
import math

scene_background(색상['검정'])

중심 = sphere(
    pos=vector(0, 0, 0),
    radius=0.8,
    color=색상['노랑'],
    emissive=True
)

print("무지개 궤도 정원 시작")
```

학생에게 보여줄 말:

> 처음부터 완성하려고 하지 않습니다. 이름을 붙일 수 있는 작은 장면 하나만 만들고 프로젝트로 저장합니다.

### 3.2 새 프로젝트 만들기

화면에서 `프로젝트` 버튼을 누르고 `새 프로젝트 만들기`를 선택한다.

입력값:

| 필드 | 예시 |
| --- | --- |
| 프로젝트 이름 | 무지개 궤도 정원 |
| GitHub 저장소 이름 | `vpylab-project-212348`처럼 자동 제안된 값을 그대로 사용 |
| 설명 | 색, 반복, 삼각함수를 이용해 움직이는 정원을 만드는 프로젝트 |

기대 결과:

- 1-3초 안에 VPyLab 프로젝트 카드가 만들어지고 바로 열릴 수 있다.
- 카드에는 처음에는 `GitHub 준비 중`이 보일 수 있다.
- 한글 프로젝트 이름은 화면과 README에는 그대로 보이고, GitHub 저장소 이름만 안전한 영문/숫자 형태로 만들어진다.
- GitHub 저장소와 Pages 실행 페이지가 준비되면 카드에 `GitHub`, `Pages`, `커밋` 링크가 나타난다.
- 준비 중에 저장을 눌러도 VPyLab 이력에는 먼저 저장된다.

관찰 포인트:

- 학생 입장에서는 "프로젝트가 열린 뒤에도 뒤에서 준비되는 것"으로 이해시키는 편이 좋다.
- GitHub 준비가 늦어져도 코드를 잃어버리는 흐름이면 안 된다.

### 3.3 첫 자세히 저장

프로젝트를 연 뒤 저장 버튼을 누른다. 첫 저장은 자세히 저장으로 안내된다.

입력값:

| 필드 | 예시 |
| --- | --- |
| 제목 | 첫 장면 만들기 |
| 한 일 | 검은 배경에 노란 중심 구를 만들고 프로젝트 이름을 정했다. |
| 막힌 점 | 아직 공이 움직이지 않는다. |
| 다음 할 일 | 여러 색 공을 원형으로 배치하고 회전시키기 |

예상 항해 일지:

```markdown
# 항해 일지: 2026-05-08

## 10:12 - 첫 장면 만들기

- 작성자: 김서연
- 한 일: 검은 배경에 노란 중심 구를 만들고 프로젝트 이름을 정했다.
- 막힌 점: 아직 공이 움직이지 않는다.
- 다음 할 일: 여러 색 공을 원형으로 배치하고 회전시키기
- 코드: 14줄
```

학생에게 보여줄 말:

> 자세히 저장은 보고서 쓰기가 아닙니다. 미래의 내가 다시 시작할 수 있을 정도로만 적으면 충분합니다.

### 3.4 기능 추가 후 빠른 저장

학생은 공을 여러 개 만들고 색을 바꾸는 작은 변경을 한다.

```python
from vpython import *
import math

scene_background(색상['검정'])

중심 = sphere(pos=vector(0, 0, 0), radius=0.8, color=색상['노랑'], emissive=True)

공들 = []
for i in range(8):
    각 = i * math.pi / 4
    공 = sphere(
        pos=vector(3 * math.cos(각), 0, 3 * math.sin(각)),
        radius=0.25,
        color=무지개[i % 7],
        make_trail=True
    )
    공.각 = 각
    공들.append(공)

print("무지개 공 8개 배치")
```

빠른 저장 입력값:

| 필드 | 예시 |
| --- | --- |
| 제목 | 무지개 공 8개 배치 |

예상 항해 일지 추가:

```markdown
- 10:25 무지개 공 8개 배치, 코드 +12줄, 김서연
```

학생에게 보여줄 말:

> 빠른 저장은 게임의 체크포인트입니다. 작은 성공이 생겼을 때 짧게 남깁니다.

### 3.5 움직임 추가 후 자세히 저장

학생은 회전 애니메이션을 추가한다.

```python
from vpython import *
import math

scene_background(색상['검정'])

중심 = sphere(pos=vector(0, 0, 0), radius=0.8, color=색상['노랑'], emissive=True)

공들 = []
for i in range(8):
    각 = i * math.pi / 4
    공 = sphere(
        pos=vector(3 * math.cos(각), 0, 3 * math.sin(각)),
        radius=0.25,
        color=무지개[i % 7],
        make_trail=True
    )
    공.각 = 각
    공.속도 = 0.01 + i * 0.002
    공들.append(공)

while True:
    rate(60)
    for 공 in 공들:
        공.각 += 공.속도
        공.pos = vector(3 * math.cos(공.각), 0, 3 * math.sin(공.각))
```

자세히 저장 입력값:

| 필드 | 예시 |
| --- | --- |
| 제목 | 공 회전 애니메이션 추가 |
| 한 일 | 각 공에 각도와 속도 값을 저장하고 while 루프에서 위치를 계속 갱신했다. |
| 막힌 점 | 처음에는 모든 공이 같은 속도로 돌아서 덜 자연스러웠다. |
| 다음 할 일 | 중심 정원에 기둥과 꽃잎을 추가하기 |

예상 항해 일지 추가:

```markdown
## 10:48 - 공 회전 애니메이션 추가

- 작성자: 김서연
- 한 일: 각 공에 각도와 속도 값을 저장하고 while 루프에서 위치를 계속 갱신했다.
- 막힌 점: 처음에는 모든 공이 같은 속도로 돌아서 덜 자연스러웠다.
```

## 6. 기능 검증 체크리스트

시연하면서 아래 항목을 확인한다.

- 새 프로젝트를 만들 때 VPyLab 프로젝트 카드가 먼저 생성된다.
- GitHub 준비 중 상태에서도 프로젝트를 열고 코드를 계속 수정할 수 있다.
- GitHub 준비 실패 또는 미연결 상태에서는 소유자가 `GitHub 연결` 또는 `다시 연결`을 눌러 저장소 생성을 다시 시작할 수 있다.
- GitHub 준비가 끝나면 프로젝트 카드에 저장소와 Pages 링크가 나타난다.
- 첫 저장 또는 30분 경과 후 저장에서는 자세히 저장 안내가 나온다.
- 빠른 저장은 제목 한 줄만 요구한다.
- 자세히 저장은 제목과 한 일을 필수로 요구한다.
- 저장 성공 후 `main.py`, `index.html`, `history.md`, `docs/voyage/YYYY-MM-DD.md`가 갱신된다.
- Pages 링크를 열면 최신 코드가 실행된다.
- README 소개글을 쓸 때 항해 일지의 내용만으로도 "만든 과정", "막힌 점", "다음 계획"을 채울 수 있다.

## 7. 현재 구현에서 주의할 점

- 프로젝트 생성은 현재 VPyLab 공간을 먼저 만들고 GitHub 저장소와 Pages는 백그라운드에서 준비한다. 그래서 학생에게는 "열리면 바로 작업해도 된다"고 안내하는 편이 맞다.
- 프로젝트 제목은 한글로 써도 된다. 단, GitHub 저장소 URL은 공유 안정성을 위해 자동으로 영문/숫자 slug가 된다.
- GitHub 연결이 실패하거나 아직 없는 프로젝트는 프로젝트 카드에서 소유자가 다시 연결할 수 있다. 이때 저장소 이름은 기본값을 그대로 쓰거나 영문/숫자/하이픈으로 수정한다.
- GitHub collaborator 권한은 VPyLab 초대 코드와 별도다. 팀원이 직접 GitHub에 커밋하려면 저장소 소유자가 GitHub collaborator로 초대해야 한다.
- VPyLab 프로젝트를 삭제해도 GitHub 저장소는 자동으로 삭제되지 않는다. 발표 링크와 포트폴리오를 보존하기 위한 기본 정책이다.
- README를 학생이 앱 안에서 직접 편집하는 전용 화면은 아직 분리되어 있지 않다. 현재 본보기에서는 GitHub README를 다듬는 단계로 설명하거나, 향후 VPyLab 안에 "소개글 작성" 화면을 추가하는 것이 자연스럽다.
- 가이드의 "완성 버튼" 흐름은 설명에는 있지만 프로젝트 UI에는 아직 명확한 버튼으로 연결되어 있지 않다. 학생 시연에서는 현재 저장과 README 작성 흐름을 중심으로 보여주는 것이 안전하다.
