# VPyLab 콘텐츠 리뉴얼 메모

작성일: 2026-04-24

## 리뉴얼 방향

VPyLab의 문제는 두 역할을 분리한다.

- 채점형 코어: 컴퓨팅 사고력의 핵심 개념을 정확히 확인한다.
- 튜토리얼형 예제: 음악, 미디어아트, 수학, 과학, 창작 영역에서 실행하고 변형하며 감각을 얻는다.
- 갤러리 활동: 정답 대신 리믹스, 출처, 변경 설명, 창의적 변형을 공유한다.

## CT 채점형 코어

CT는 학생이 프로그래밍 개념을 실제로 익혔는지 확인하는 영역이다. 단순 정답 맞히기가 아니라, 시작 코드에 목표와 빈칸을 두고 학생이 한두 줄씩 완성할 수 있게 한다.

- CT-1: `sphere()`, `pos`, `radius`, `color`, `emissive`
- CT-2: 객체 변수의 속성 변경과 순차 실행
- CT-3: `def` 함수 정의와 재사용 호출
- CT-4: `if/elif/else`와 `for range`
- CT-5: 리스트에 객체와 속도를 저장하고 반복문에서 상태 업데이트
- CT-6: 변수를 먼저 정하고 객체 생성에 연결
- CT-7: `range()` 숫자를 조절해 반복 횟수 이해
- CT-8: 함수 매개변수를 실제 도형 속성에 연결
- CT-9: 온도 값을 색으로 바꾸는 조건문
- CT-10: 두 리스트를 같은 인덱스로 연결해 위치 업데이트
- CT-11: `for`와 리스트 인덱스를 연결해 색 꺼내기
- CT-12: 독립적인 `if` 두 개가 모두 적용되는 구조
- CT-13: `if else`로 둘 중 하나 선택
- CT-14: `if elif else`로 세 단계 선택
- CT-15: `if elif elif else`로 네 단계 선택
- CT-16: `list.append()`로 반복문에서 만든 객체 모으기
- CT-17: `random.uniform()`, `random.choice()`로 랜덤 위치와 색 만들기

이 영역은 자동 채점이 학생에게 명확한 피드백을 주는 것이 중요하므로, 정확한 `assertions`를 유지한다. 일부 문제는 `codeChecks`를 함께 사용해 함수/반복/리스트 접근 같은 개념 사용도 확인한다.

## 튜토리얼형 영역

CT 외 영역은 정답 좌표 맞추기보다 "실행해 보고 바꿔 보기"를 우선한다. `gradeType: run`은 학생이 코드를 실행해 실제 장면을 만들었는지만 확인하고, 세부 결과는 갤러리와 토론으로 확장한다.

- 수학/과학: 공식 암기보다 변수 하나를 바꾸며 결과를 관찰한다.
- 창작/미디어아트: 완성 예제를 리믹스하며 색, 움직임, 조명, 리듬을 바꾼다.
- 음악: `음표()`, `악기()`, `화음()`, `효과음()`, `배경음악()`을 한글 API 중심으로 노출한다.
- 문제 수는 카테고리별 균등 분량보다 교육적 밀도를 우선한다. 특히 음악과 미디어아트는 더 많이 추가해도 된다.

## 오픈소스 문화 연결

문제 설명과 코드 주석은 다음 행동으로 자연스럽게 이어지도록 설계한다.

- 원작 아이디어를 남기고 변형 포인트를 설명한다.
- 색, 리듬, 개수, 공식, 속도처럼 작은 변경을 리믹스의 단위로 삼는다.
- 갤러리에 올릴 때 "무엇을 바꿨는가"를 설명하도록 유도한다.
- 한글 API를 먼저 보여주고 영어 API는 문서에서 병기해, 초심자 친화성과 오픈소스 호환성을 함께 챙긴다.

## 다음 개선 후보

- 갤러리 제출 화면에 "원작", "변경한 점", "배운 점" 필드를 명시한다.
- 튜토리얼 미션 완료 후 "리믹스해서 갤러리에 올리기" 버튼을 연결한다.
- 음악/미디어아트 예제에 짧은 미리보기 썸네일 또는 태그를 추가한다.
- CT-5는 고1 초심자에게 어려울 수 있으므로, 필요하면 "리스트 만들기"와 "리스트 업데이트" 두 단계로 나눈다.
- 음악/미디어아트는 학생 갤러리 반응을 보며 계속 추가한다.
