# VPyLab 콘텐츠 점검 메모

작성일: 2026-04-24

## 방향성

VPyLab의 채점은 학생이 핵심 프로그래밍 개념을 이해했는지 확인하는 장치로 두고, 갤러리는 창의적 변형과 오픈소스식 공유를 경험하는 장으로 둔다.

- 채점형 미션: 주로 CT(컴퓨팅 사고)에서 함수, 반복문, 조건문, 리스트, 상태 업데이트를 확인
- 튜토리얼형 예제: 음악, 미디어아트, 수학, 과학에서 완성 코드 관찰과 안전한 변형 제공
- 갤러리형 챌린지: 정답 없이 리믹스, 출처, 설명, 변형 포인트를 중심으로 공유

## 현재 미션 분포

현재 `client/src/data/missions.js`에는 30개 미션이 있으며, 카테고리별 5개씩 배치되어 있다.

| 카테고리 | 현재 성격 | 채점 상태 | 조정 제안 |
| --- | --- | --- | --- |
| CT | 기본 개념과 상태 업데이트 | 5개 모두 A 채점 | 채점형 코어로 유지 |
| CR | 도형 조립 창작 | 5개 모두 A 채점 | 일부는 튜토리얼/갤러리형으로 이동 |
| MA | 좌표, 삼각함수, 함수 그래프 | 5개 모두 A 채점 | 튜토리얼형 탐구 예제로 전환 |
| SC | 물리 운동 시뮬레이션 | 5개 모두 A+B 채점 | 궤적 채점은 줄이고 관찰형으로 전환 |
| AR | 코드 아트/미디어아트 | 4개 A, 1개 run | 정답형보다 변형형 예제로 확대 |
| SN | 사운드 코딩 | notes/code/A+B 혼합 | 음악 창작 튜토리얼과 갤러리형 강화 |

## 핵심 문제

1. CT 외 영역도 대부분 정답 좌표/정답 도형을 요구한다. 음악, 미디어아트, 수학, 과학은 학생에게 "창작"보다 "예쁜 채점 문제"처럼 보일 수 있다.
2. 음악과 미디어아트가 프로젝트 정체성에 비해 아직 얇다. `음표()`, `화음()`, `악기()`, `음계`, `무지개`, `파스텔` 같은 학생 친화 API를 쓰는 예제가 더 필요하다.
3. 일부 기능은 문서에는 있거나 미션에서 쓰이지만 커리큘럼 노출이 약하다. 예: `arrow`, `compound`, `local_light`, `distant_light`, `악기()`, 차트 한글 별칭.
4. 오픈소스 문화 학습은 기능보다 흐름이 중요하다. 갤러리 제출, 리믹스, 원작 링크, 변경 설명, 라이선스 동의, 리뷰 코멘트를 미션 후속 활동으로 연결해야 한다.

## 재분류 제안

### 채점형 코어

- CT-1: 3D 객체 생성
- CT-2: 속성 변경과 순차 실행
- CT-3: 함수 정의와 재사용
- CT-4: 조건문과 반복문
- CT-5: 리스트와 상태 업데이트. 단, 난이도가 높으므로 선택 도전 또는 두 단계 분리 권장

### 튜토리얼형으로 이동 권장

- CR-1~CR-5: 캐릭터/집/태양계 조립은 갤러리형 변형이 더 적합
- MA-2~MA-5: 정다각형, 나선, 함수 그래프, 피보나치는 `N`, `R`, 색상 공식을 바꾸는 탐구형이 적합
- SC-1~SC-5: 궤적 채점보다 중력, 속도, 반발 계수, 초기 조건을 바꾸는 관찰형이 적합
- AR-1~AR-5: 색상/파동/입자/무대 연출 중심의 미디어아트 스튜디오로 재구성
- SN-1~SN-5: 음표 순서 확인은 최소화하고 리듬, 화음, 악기, 효과음 장면 만들기로 확장

## 음악 예제 후보

- 세 음으로 나만의 알림음 만들기: `음표()`, `sleep()`
- 반복문으로 리듬 패턴 만들기: 리스트와 반복
- 도-미-솔 화음 분위기 바꾸기: `화음()`, `음계`, `높은음계`, `낮은음계`
- 악기 바꿔 듣기: `악기("피아노", ...)`, `악기("칩튠", ...)`
- 게임 장면 효과음 디자인: `효과음("jump")`, `효과음("coin")`, `효과음("success")`
- 8비트 BGM 위에 멜로디 얹기: `배경음악()`, `음표()`, `배경음악정지()`

## 미디어아트 예제 후보

- 색이 숨 쉬는 구름: `파스텔`, `opacity`, `sin`
- 파동으로 춤추는 바닥: `무지개`, `음계`, `sphere` 격자
- 별자리 생성기: `scene_background(색상['검정'])`, 랜덤 별, 선 연결
- 파티클 붓: 마우스/시간 기반 입자 흔적
- 음악 조명 무대: `배경음악()`, `효과음()`, 색상 변화
- 나만의 VJ 루프: 반복 애니메이션을 갤러리에 공유

## 수학/과학 융합 예제 후보

- 정다각형 생성기: 꼭짓점 수 `N`을 바꾸며 관찰
- 피보나치 꽃밭: 황금각을 바꿔 패턴 비교
- 함수 지형 만들기: `sin`, `cos`, 색상맵 변형
- 다른 행성의 중력: `g` 값을 바꿔 낙하 비교
- 튕기는 공 실험실: 반발 계수와 에너지 손실 관찰
- 궤도 실험실: 초기 속도를 바꿔 충돌/공전/탈출 비교

## 문서/API 원칙

- 학생용 Python 예제에서는 공식 한글 API를 우선 노출한다.
- 영어 API는 병기하되, 초심자용 설명에서는 한글 이름을 먼저 둔다.
- `rate()`, `sleep()`, `음표()`, `악기()`는 편집기가 자동으로 `await` 처리하므로 예제에서는 그대로 쓴다.
- 미션 작성 문서에는 `소리()`, `음표()`, `화음()`, `효과음()`, `배경음악()`, `배경음악정지()`, `악기()`, `음계`, `무지개`, `파스텔`을 공식 API로 명시한다.
